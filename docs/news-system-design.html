Node 4: TAVILY指示 - LLMプロンプト（イテレーション内）

Model: gpt-4o-mini（確定）

[System] あなたは、与えられた「リサーチトピック」に対して、Tavily検索に適した新規性の高い検索クエリを1件だけ作成する検索アシスタントです。 今日の日付: {{本日の日付.result}} ## ゴール 短く明確でTavilyでの検索に適した1件のクエリを生成する ## 制約 - 本日の日付に近い、最新の情報を取得する - 最大30文字以内 - 過去に使用されたクエリと重複または類似しない - 内容は具体的で、検索意図がはっきりしていること（対象、条件、期間などを必要に応じて含める） - リサーチトピックと同じ言語で作成する - 出力はクエリ文字列のみとし、説明文や装飾は含めない - 必ず1件のみを出力する（複数行やリスト形式は禁止） [User] 以下の情報をもとに、Tavily検索用の最新の情報が取得できるよう新規検索クエリを1件だけ作成してください。 ## リサーチトピック {{item}}
効果: 「本日の日付」ノードから実行時の日付を受け取ることで、LLMの知識カットオフに関係なく正しい年号でクエリを生成できる。例: 「Claude Code」→「Claude Code 2026年3月 新機能」

Node 5c: 結果結合 - コード（イテレーション内・最終ノード）

並列実行されたニュース検索（5a）とYouTube検索（5b）の出力を1つの配列に結合する。

重要: Difyのイテレーションでは、出力は最終ノード1つからのみ取得される。並列ノード（5a・5b）が終端だとイテレーション出力がnullになるため、必ずこの結合ノードをイテレーションの最後に配置すること。
def main(arg1: list, arg2: list) -> dict: results = [] if arg1 and isinstance(arg1, list): results.extend(arg1) if arg2 and isinstance(arg2, list): results.extend(arg2) return {"result": results}
項目値
入力変数arg1（Array）= Node 5a ニュース検索結果、arg2（Array）= Node 5b YouTube検索結果
出力変数result（Array）- 両方の検索結果を結合した配列

Node 6: 結果マージ + 重複排除 - コード（イテレーション後）

イテレーションの全出力を統合し、URL重複排除・スコア順ソート・上位15件を抽出する。

注意: イテレーション出力は Array[String] 型で渡される場合がある。各要素を json.loads でパースする必要がある。また、Tavily応答は results キーの下にネストされている場合があるため、batch.get('results', []) でアクセスする。
def main(arg1: list) -> dict: import json all_results = [] seen_urls = set() for batch in arg1: if not batch: continue if isinstance(batch, str): try: batch = json.loads(batch) except Exception: continue if isinstance(batch, dict): items = batch.get('results', []) elif isinstance(batch, list): items = batch else: continue for item in items: if not isinstance(item, dict): continue url = item.get('url', '') if url and url not in seen_urls: seen_urls.add(url) is_yt = 'youtube.com' in url all_results.append({ 'title': item.get('title', ''), 'url': url, 'content': item.get('content', '')[:200], 'published_date': item.get('published_date', ''), 'score': item.get('score', 0), 'type': 'youtube' if is_yt else 'article' }) # スコア順にソートして上位15件 all_results.sort(key=lambda x: x.get('score', 0), reverse=True) return {"result": json.dumps(all_results[:15], ensure_ascii=False)}
項目値
入力変数arg1（Array）= イテレーションの出力配列
出力変数result（String）- JSON文字列（最大15件）

Difyコードサンドボックスの注意事項

実装時のハマりポイント: Difyのコードノード（Python）には以下の制約がある。
注意事項詳細
def main() は行頭から コードをペーストした際に先頭に空白が入ると IndentationError になる。必ず def が左端（列0）から始まっていることを確認
import は def main() 内に書く トップレベルに import を書くとインデントエラーになる場合がある。def main() の中で import json のように書くのが安全
from datetime import datetime は使用不可 Difyサンドボックスでは動作しない。代わりに import datetime + datetime.datetime.now() を使用
イテレーションの最終ノードは1つだけ 並列ノードがイテレーションの終端になると出力がnullになる。必ず結合用コードノードを最後に配置
出力変数名はDify UIと一致させる コード内の return {"result": ...} のキー名が、Dify UIで設定した出力変数名と一致しないと「Output result is missing」エラーになる

Dify API コール仕様

Python スクリプトから Dify Advanced Chat API を呼び出す際のリクエスト・レスポンス仕様

リクエスト

POST https://api.dify.ai/v1/chat-messages Headers: Authorization: Bearer {DIFY_API_KEY} Content-Type: application/json User-Agent: DUD-NewsAggregator/1.0 ← Cloudflare制限回避に必要 Body: { "inputs": {}, "query": "AI,Claude Code,今井翔太", "response_mode": "blocking", "conversation_id": "", "user": "hp-news-fetcher" }
注意: Cloudflare がデフォルトの Python User-Agent をブロックするため、User-Agent: DUD-NewsAggregator/1.0 を必ず設定すること。

レスポンス

{ "event": "message", "message_id": "xxx", "conversation_id": "xxx", "answer": "[{\"id\":1, ...}]", ← ニュースJSON（LLM応答テキスト） "created_at": 1709500000 }
LLMの応答形式: answer フィールドにJSON配列が返されるが、マークダウンのコードブロック（```json ... ```）で囲まれている場合がある。スクリプト側でコードブロックを除去してからパースする必要がある。

Pythonスクリプトの実装

cronスクリプトと設定ファイル（独立リポジトリ dud_news_aggregator で管理）

scripts/fetch_news.py

#!/usr/bin/env python3 """ニュース自動取得スクリプト（cron実行用） Dify Advanced Chat API経由でTavilyニュースを取得し、JSONに保存する。 エラー時は既存データを上書きしない安全設計。""" import json, os, shutil, sys from datetime import datetime from urllib.request import Request, urlopen # config読み込み from config.config import ( DIFY_API_URL, DIFY_API_KEY, NEWS_GENRES, NEWS_MAX_ITEMS, NEWS_FETCH_TIMEOUT, ) def fetch_from_dify() -> str: """Dify Advanced Chat APIを呼び出してニュースを取得""" payload = json.dumps({ "inputs": {}, "query": NEWS_GENRES, "response_mode": "blocking", "conversation_id": "", "user": "hp-news-fetcher", }).encode("utf-8") req = Request(DIFY_API_URL, data=payload, headers={ "Content-Type": "application/json", "Authorization": f"Bearer {DIFY_API_KEY}", "User-Agent": "DUD-NewsAggregator/1.0", }, method="POST") with urlopen(req, timeout=NEWS_FETCH_TIMEOUT) as resp: body = resp.read().decode("utf-8") data = json.loads(body) answer = data.get("answer") if not answer: raise RuntimeError("No answer in Dify response") return answer def validate_news(raw_answer: str) -> list: """LLM応答からJSON配列を抽出・バリデーション""" text = raw_answer.strip() if text.startswith("```"): # マークダウンコードブロック除去 lines = text.split("\n")[1:] if lines and lines[-1].strip() == "```": lines = lines[:-1] text = "\n".join(lines) news = json.loads(text) # 必須フィールドチェック required = ["id", "date", "category", "title", "link"] for field in required: if field not in news[0]: raise RuntimeError(f"Missing required field: {field}") return news def backup_existing(): """既存ファイルを日時付きでバックアップ（毎回保存）""" timestamp = datetime.now().strftime("%Y-%m-%d_%H%M%S") backup_file = f"data/tech_news_{timestamp}.json" shutil.copy2(TARGET_FILE, backup_file) def save_json(news: list): """JSONをアトミックに書き込む（一時ファイル → os.replace）""" tmp_file = TARGET_FILE + ".tmp" with open(tmp_file, "w") as f: f.write(json.dumps(news[:NEWS_MAX_ITEMS], ensure_ascii=False, indent=2)) os.replace(tmp_file, TARGET_FILE)

コスト試算

1日2回実行、3ジャンルの場合

Tavily API（ニュース + YouTube） 3ジャンル × (5件+3件) × 2回/日 = 16リクエスト/日 → 約480リクエスト/月（Basicなら480クレジット。Advancedならニュース分が倍で約780クレジット）
Dify LLM（クエリ最適化） gpt-4o-mini: 3ジャンル × 2回/日 = 6回/日 → 約180回/月（ごく少量・低コスト）
Dify LLM（整形用） 1回あたり約2,000トークン × 2回/日 → 約120,000トークン/月
Dify Cloud 無料プランまたはSandboxプランで対応可能
合計 ほぼ無料（Basic設定なら月480クレジットでTavily無料枠1,000回に十分収まる）

  1. tavily.com でアカウント作成、APIキーを取得
  2. udify.app にログイン → ワークスペース設定でTavilyツールを追加

  5. テスト実行で正しいJSONが出力されることを確認
